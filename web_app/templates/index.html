算法介绍

定义: 带容量约束的车辆路径问题 (Capacitated Vehicle Routing Problem, CVRP) 是指一定数量的客户有不同数量的货物需求，配送中心安排车辆向客户供货，并组织适当的行车路线，目标是找出最佳的路线，使得在满足所有客户需求和车辆容量限制的条件下，总行驶距离最短。

核心要素:

  • 仓库 (Depot): 所有路线的起点和终点。
  • 客户 (Customers): 需要被服务的点，有各自的需求量。
  • 车辆 (Vehicles): 每辆车有固定的最大载货容量。
  • 约束 (Constraints):
    • 每条路线的总需求量不能超过车辆容量。
    • 每个客户必须被服务一次。
  • 目标 (Objective): 最小化所有车辆行驶的总距离。

来源: 由G. Clarke和J.W. Wright于1964年提出，是一种经典的启发式算法。

原理: 核心思想是“节省”。初始时，每个客户都由一辆专用车辆服务。然后，算法计算合并任意两条路线能节省的距离，并优先合并节省值最大的路线对。

运作过程:

  1. 为每个客户创建一条从仓库出发的独立路线。
  2. 计算所有客户对 (i, j) 的节省值 S(i, j) = d(0, i) + d(0, j) - d(i, j)。
  3. 将节省值从大到小排序。
  4. 按顺序遍历节省值列表，若合并路线不违反车辆容量约束，则执行合并。
  5. 重复直到没有更多可合并的路线。

适用场景: 适合求解中小型VRP问题，计算速度快，易于实现，能快速给出质量不错的解。

来源: 由Marco Dorigo于1992年提出，灵感来源于蚂蚁寻找食物的行为。

原理: 模拟蚂蚁通过释放“信息素”来寻找最短路径。蚂蚁倾向于选择信息素浓度更高的路径，而较短的路径信息素积累更快，形成正反馈，引导整个蚁群找到优解。

运作过程:

  1. 初始化所有路径上的信息素。
  2. 让一组“蚂蚁”根据信息素浓度和距离概率性地构建路线。
  3. 根据蚂蚁构建的路线总长度更新信息素，短路径获得更多信息素。
  4. 所有路径上的信息素会以一定速率挥发。
  5. 重复构建和更新过程，直到达到迭代次数。

适用场景: 适用于复杂的组合优化问题，鲁棒性好，能找到高质量的解，但计算成本相对较高。

参数说明:

  • num_ants： 蚂蚁数量，影响搜索广度。
  • alpha： 信息素重要程度，越大越依赖已有信息素。
  • beta： 启发函数权重，越大越依赖路径距离。
  • rho： 信息素挥发系数，控制旧信息素的保留程度。
  • iterations： 算法迭代次数，越多越可能找到优解。

来源: 基于达尔文的自然选择和遗传学原理，由John Holland在1970年代提出。

原理: 模拟生物进化过程，通过“选择”、“交叉”和“变异”等操作，使代表解的“种群”一代代进化，最终收敛到最优或次优解。

运作过程:

  1. 创建初始种群，每个“个体”是一套完整的VRP路线。
  2. 根据总距离评估每个个体的“适应度”。
  3. 选择适应度高的个体进行“交叉”（交换部分路线）以产生后代。
  4. 对后代进行小概率的“变异”（如交换两个客户的顺序）。
  5. 新产生的后代替换种群中适应度较低的个体。
  6. 重复此过程，直到达到预设的代数。

适用场景: 强大的全局搜索能力，适合求解大规模、复杂的优化问题，但参数设置较为敏感，计算时间较长。

  • population_size： 种群数量，影响解空间覆盖范围。
  • crossover_rate： 交叉概率，决定个体交叉生成新解的频率。
  • mutation_rate： 变异概率，用于维持种群多样性，防止陷入局部最优。
  • num_generations： 迭代次数（代数），越多越可能获得更优解。
  • selection_method： 选择方式，如轮盘赌、锦标赛选择等，决定哪些个体能进入下一代（本项目默认选择轮盘赌法）。

来源: 一种非常直观和古老的贪心算法。

原理: 贪心策略。从一个点（通常是仓库）开始，总是选择离当前位置最近且尚未被访问的客户作为下一个访问点，直到车辆容量用尽或所有客户被服务。

运作过程:

  1. 从仓库开始一条新路线。
  2. 找到离当前节点最近且未被访问的客户。
  3. 如果将该客户加入当前路线不违反容量限制，则将其加入，并移动到该客户位置。
  4. 如果容量不足，则返回仓库，开启一条新路线。
  5. 重复此过程，直到所有客户都被分配到路线中。

适用场景: 算法非常简单快速，适合作为其他复杂算法的初始解生成器，或在对解的质量要求不高但对速度要求极高的场景。其解的质量通常不高。
